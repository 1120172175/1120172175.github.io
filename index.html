说说这两天的面试

发表于 2019-10-30

date: 2019-10-30 22:24:51

这两天忙于找实习以及面试去了，原来项目的进度都没有去推了，博客已经一周没有更了，但是这毕竟是一个过程嘛。

算一算，一个电话面试，一个在线笔试，三个现场面试。第一个电话面试比较一般吧，可能是对方要求比较高一些，于是乎自我感觉良好下挂掉了。在线笔试那也是石沉大海。主要是三场现场面试，第一场还行，原来以为那30道选择题答得不好应该没戏，但是后来居然继续面了，应该是一个小boss面的我，四十来岁的一个大叔，感觉大叔还比较笑嘻嘻的，居然没问啥nlp反而问了很多聊天软件实现的细节，真的是凑巧填上去的经历居然成为面试官主要问的问题，最后大叔也是笑嘻嘻的愿意要我了。还是蛮高兴的，毕竟人生第一次这样的工作面试。第二场面试就比较那啥了，一个刚毕业的小姐姐面的我，感觉很不上道，自己都不太知道问啥，然后最后嫌我菜就没要，就感觉不同的人看人的角度也是很不一样的，面试真的很看面试官的。然后说说今天下午那场吧，也是我最满意的一场，公司比较大然后给人感觉也比较正规，流程都还行。面试官也说不重视项目经验，毕竟才大三，然后让我讲自己的一次类似于解决问题的难忘经历那种，我就讲了当时做数字逻辑大作业时候的心路历程，最后面试官也还是很满意的吧，对我印象也是很不错，所以这个应该是势在必得，不用很担心的那种，如果这个ok的话就可以拒了iyuke的实习了。

总的来说，面试真的随缘，面试官可能就比较喜欢你，也可能就对你不感兴趣。

昨日的YihooTalks

发表于 2019-10-28

date: 2019-10-28 08:12:51

昨天晚上的YihooTalks很棒，真的很棒！

果然，每次我计划好了一定要去的事情总会在临走前想着还是不去了吧，但是事实证明，几乎每次这样的事情我都去了，这样的活动我都参加了，而且没有一次是让我失望的。好的，说回昨天的YihooTalks吧。

首先说一下昨天的心路历程，昨天其实一开始并没有感觉很棒，第一个演讲讲的是亲子关系，感觉略微不对胃口。第二个演讲讲的是消费主义下的啥，整个演讲，作为一个观众去评价的话还是稍有欠缺的，不够吸引人，论点也不够成熟，背稿的感觉也是挺突出的，但是这丝毫不影响接下来的精彩~

接下来让我印象稍微深刻一点的有乔文作的信息论相关的演讲，“我讲清楚了吗”这中询问的形式早在知乎就了解过，却从未真正践行这样的习惯，昨天在Yihoo可谓是再一次唤醒了这个角度的交流技巧与对他人的尊重。一句话，从中解读多种意思，或者能够看出其中的深层次的意思和背后的逻辑，着实是需要人去学习的。

再说说行业调研演讲中的收获吧，让我最感兴趣、印象最深的就是张琳梓的咨询行业的调研了，在互联网+和大数据的时代，中国咨询行业的发展是否能借助中国互联网行业的迅猛发展而弯道超车呢？这个问题就先留在这里。为什么提到这个呢，因为我原初的想法是去往大数据的这个方向走，进而再往商科去靠近，仿佛这个才是自己心里想要的东西。但是实际上是什么样的呢？无从得知，可能只是你自己的一种并没有那么正确的倾向罢了。

接着，我产生了一个更大的想法。那就是重新思考，假如我明年大四休学，我在休学这一年的时间里可以干什么呢？我可以去进行行业调研，进行相关行业的实习试水和了解，完全由自己的兴趣去驱动，让自己在一年时间内去了解多个相关行业的发展和现状以及自己对其感兴趣程度有多大。到这里，我原先不成熟的休学计划现在就已经具备了更强大的理论基础。我越来越意识到这样一个事情对自己的价值了，既然我毕业直接工作，那么成为一个合格社会人自然是我需要重视的一个点，这也是芝兰的核心目的之一。老总在Yihoo的导师对话中说到迈入社会后很重要的几个点，那就是：合作、组织、行动、人际。包括但不限于这几点，所以我也想说说，针对这几点我是怎么想的，如何去论证我的休学想法的合理性。

首先是和做，在休学的一年中，我会去我的老本行nlp方向实习，等到明年这个时候我相信我自己的专业技能肯定是能够轻松找到一份还可以的实习了，这个在工作的过程中积累合作、工作的经验，同时也积累一些旅行的花销。然后就是组织，芝兰给我感觉就是一个很亲和，很有凝聚力的优秀平台，我也是相信我能加入其中的。接着是行动，这是当然，行胜于言，在一年的游学当中，很多事情都是无法预料的，先思考后行动or先行动后热爱？这些都是不得而知的，需要在整个过程中行动起来去探索，去深入，去思考。最后就是人际了，总也感觉现在的自己在人际交往上很多方面都有欠缺，既然自己将自己定位成一个不是纯搞技术的人，那么这个方面自然需要很多重视，那么在游历的这一年中，就会有很多机会去学人和习事了，纸上得来终觉浅，很多道理知乎上都有，但是却不能深入人心，还是需要去实践，去亲身经历，去学习。看到的只是信息，思考过理解过才叫知识。我越来越坚信这一年的休学游历对自己的影响是终身的。

最后还想说一说的就是做多件事的能力。忙起来的日子才充实，做多件事的能力真的很重要，日子不可能和高中一样只为学习，以后角色会有很多，工作家庭等等都会有自己的角色，如何处理好这些东西，只能够实践出真知。同时实习，上课，看书等等，多去做事情吧，不要让时间白白浪费，至少不要到晚上睡觉前感觉后悔白天没有多做事情。

好久了

date: 2019-09-22 21:59:45

好像好久好久没有写东西了，但是实际上上一次写东西是在一个月以前。这个月确实没有让自己失望，该干的事情都干得很好很满意，没有三分钟热度，保持了一个月的早起，并且已经习惯于早起了，这种感觉还是很好很好的，不需要坚持的那种，是一种自然而然的生活状态。和上一次相比，这一次的区别可能在于，不再想去和别人说了吧，好像很多事情都是这样，你想去做一件事，然后你一开始总是在摸着石头过河，难免会做不好。但是只要这份心在，总会在不久之后做得还可以。早起是这样，学习是这样，恋爱是这样，和别人打交道也是这样，处理各种各样的问题也都是这样，慢慢的，做事就成熟了，心态也稳了，没有太多波澜。

现在看来吧，如果不是真的有啥事逼着你的话，自觉地早起更应该是一种心境，而不是一种需要去坚持的行为。就好像你看书看电影看到入迷处会沉浸其中，也许会停止思考去感受其中的美妙，也许会集中精神去思考些自己与画中故事的共鸣或是感慨。好好把生活沉浸下来，留心身边那些纯粹的东西，和朋友们多聊聊天，多去吹吹北湖的风，看看北湖的鹅，多看看那些有故事的有思想的有温度的书，多去认真思考如何把一件事情做好，这样的事情还有很多吧！这种心境是值得多去修炼的，生活到一种程度了，这些东西自然就会来，强求不得。那么既然来了，就要好好去享受一个人感受窒息奋斗的畅快，一个人把心放在深沉之中的温柔，一个人去向着梦想朝圣的虔诚。

生活很美，你自己也是。虽然最后一句话很不应景，但我还是要说，那就是，我现在最恨别人背叛我。

好好生活，这样你才能为所欲为。

暑假这些天

发表于 2019-08-23

date: 2019-08-23 23:13:20

暑假马上也就要接近尾声了，虽然感觉这个暑假没有干很多事情，但还是有很多收获的，毕竟也是更了解自己了。

其实吧，很多时候一个人并不了解自己。为什么这么说呢？因为很多时候遇到烦心的事情或者感觉很不好的时候并不知道怎么样去有效地解决问题。啥也不想瘫着只等把问题暂停罢了，或者说虽然逃避了问题，但是并没有把心里那口气真正地舒出来。真的好难呀。

原来在宿舍待久了真的会很堕落，我一直以为我肯定不会因为在宿舍待久了而堕落，但事实证明我错了。没代码写，没有需要显示屏的硬活干的时候待在宿舍真的就容易消磨意志。太舒适了，太嘈杂且随意了。想做事情了，就到图书馆去吧，恢复你大一时候的优良习惯吧！

还有就是，真的得少看手机，知乎是必须要卸载的，b站也要少看才对，容易上瘾的东西就是这样。常备一本书去看，沉静下来想事情，做计划，然后一项一项划掉。

长时间的学习不同于学期中的学习。长期的学习最重要的不是你一天干的有多少，更重要的是节奏。一定不能灵活性较强的暑假一开始大搞特搞，后面就很容易崩，特别是自己一个人搞的时候。容易感觉很孤独孤立无趣，这种节奏是很危险的，不可持续的。一定要留足时间去适应，至少最开始要这样，不能一开始就很push自己，长跑一定要有一个合适的节奏去保持才行，开始用力过猛就会出问题。

需要长期坚持做的事情尽量在一天最开始的时候去做，比如学托福，记单词…真就是well begun，half done

今天就写这么多吧，希望好好开始你的大三。

记录一下搭建一个中文rasa-nlu踩的坑

发表于 2019-07-20

这两天可以说是都用来调模型的error了，一波未平一波又起

原始的教程参考如下网站，但是确实还是有一些地方没有说清楚的（本篇和官方教程辅助使用更佳哦）。

http://www.crownpku.com/2017/07/27/%E7%94%A8Rasa_NLU%E6%9E%84%E5%BB%BA%E8%87%AA%E5%B7%B1%E7%9A%84%E4%B8%AD%E6%96%87NLU%E7%B3%BB%E7%BB%9F.html

讲道理github上的README更良心

https://github.com/crownpku/rasa_nlu_chi

首先是安装结巴分词，这个没什么问题

$ pip install jieba

然后是安装MITIE，这个就比较坑了，需要提前安装一些东西，配一下环境才能安装成功，安装MITIE参考下面的博客(劝退没有安装vs的同志)：

https://blog.csdn.net/caroline2222/article/details/88601373

好了，现在默认你的MITIE已经安装好了，接下来就是下载人家已经训练好的类似于embedding的数据：total_word_feature_extractor_chi.dat

链接：https://pan.baidu.com/s/1kNENvlHLYWZIddmtWJ7Pdg 密码：p4vx

好了，现在来说已经下好了两个比较主要的依赖包了，然后就是rasa_nlu_chi的安装：

先创建一个文件夹比如RasaProgram，然后在这个文件夹下：

$ git clone https://github.com/crownpku/rasa_nlu_chi.git
$ cd rasa_nlu_chi
$ python setup.py install

不出意外应该就可以安装好了

然后再让我们安装很多很多很多其他的依赖包（当然不能一个一个下）
运行（所有命令行操作默认都是在rasa_nlu_chi文件夹下进行的）：

pip install -r requirements.txt

然后由于之前博客的教程是2017年的，而现在都已经9102年了，所以现在下的有些依赖包版本比较新，可能会出问题，具体的后面再说。

数据是现成的：
data/examples/rasa/demo-rasa_zh.json

然后再把之前给的那个类似于embedding的数据放到data文件夹下面。数据的准备就OK了

再然后就还要配置一下pipeline文件：
推荐使用下面的pipeline:

MITIE+Jieba+sklearn (sample_configs/config_jieba_mitie_sklearn.json):

[“nlp_mitie”, “tokenizer_jieba”, “ner_mitie”, “ner_synonyms”, “intent_featurizer_mitie”, “intent_classifier_sklearn”]

好了，现在准备工作都做好了，可以开始训练了（训练很快）

python -m rasa_nlu.train -c sample_configs/config_jieba_mitie_sklearn.yml --data data/examples/rasa/demo-rasa_zh.json --path models

这一步的话也可能会报错什么的，首先是要保证你的python之前就装好了rasa之类的，然后就是保证之前的依赖包都要装好。一定要注意！一定要注意！官方说”If you specify your project name in configure file, this will save your model at /models/your_project_name.” 不要信！因为如果给project取了名字之后就会出现各种问题，还是老老实实不要加东西吧，就–path models就可以了。
如果报错显示utf-8编码有问题之类的就看报错信息，把相应.py文件里的utf-8全换成GBK就不会报错了（我就是瞎搞的）

搭建本地nlu服务：

python -m rasa_nlu.server -c sample_configs/config_jieba_mitie_sklearn.yml --path models

最后就是测试数据了
首先输入：

curl 'http://localhost:5000/status'

看一下自己的project名字，不出意外，project名字应该是“default”，再看一下模型的名字，后面要用：
最后就是这句终极的命令了（因为到这里会各种报错）

curl -XPOST localhost:5000/parse -d '{"q":"我发烧了该吃什么药？", "project": "rasa_nlu_test", "model": "model_20170921-170911"}' | python -mjson.tool

报错①：啥 rasa_nlu_test 项目不存在，那是因为你的project名字不叫这个，如果之前查看的project名字叫default的话，就可以把这个部分的参数删掉，后面”model”:”model_**-**“是你自己的model的名字，其实就是你模型训练出来的时间，这个20170921-170911意思就是2017年9月21日17:09:11训练出来的。

报错②：”error”: “unable to initialize persistor”
这个就是我之前说的给模型命名之后很可能会出的问题…这个就是”project”:”***”这出的问题

报错③：”error”: “bad input shape (1, 5)”
这个就是没有把依赖包都安装完会出现的问题。什么？安完了还报错？那就对了。因为有一个包太新了…要降级一下。

$ pip uninstall scikit-learn
$ pip install scikit-learn==0.19.1

然后再把模型跑一遍，再搭建本地服务，再测试数据应该就没问题了
